先说原则
----

选型的三个维度：

1.**可用性**：能不能稳定访问？

2.**合规性**：会不会踩红线？

3.**性价比**：值不值得花钱？

如果你的业务对数据安全要求高，或者担心海外服务不稳定，考虑本地部署。

### 开源模型推荐

| 模型 | 参数量 | 特点 |
| --- | --- | --- |
| DeepSeek-V3 | 671B | 开源最强梯队 |
| Qwen2.5 | 72B | 中文能力强 |
| Llama 3.3 | 70B | Meta 开源，社区活跃 |
| Mistral | 各种尺寸 | 欧洲开源，效率高 |

### 部署方案

| 方案 | 硬件需求 | 适用场景 |
| --- | --- | --- |
| Ollama | 消费级显卡 | 个人开发、小规模使用 |
| vLLM | 专业 GPU | 高并发生产环境 |
| LM Studio | Mac/PC | 桌面端使用 |

---

我的选型建议
------

### 个人开发者

```
主力：DeepSeek（便宜、好用、稳定） 备选：ChatGPT/Claude（复杂任务） 本地：Ollama + Qwen2.5（离线场景） 
```

### 小团队

```
主力：DeepSeek API（性价比高） 辅力：ChatGPT Team（团队协作） 部署：vLLM + 开源模型（敏感数据） 
```

### 企业

```
云端：国产模型 API（合规优先） 私有：本地部署开源模型（数据安全） 混合：敏感数据本地，通用任务云端 
```

---

成本对比
----

以每月 100 万 tokens 调用量为例：

| 模型 | 月成本 |
| --- | --- |
| DeepSeek API | ~¥50 |
| 通义千问 API | ~¥100 |
| ChatGPT API | ~$20 （约¥145 ） |
| Claude API | ~$15 （约¥110 ） |
| 本地部署 | 电费 + 硬件折旧 |

---

大抵如此罢
-----

海外模型能用，但要有风险意识。

国产模型在追赶，差距在缩小。

本地部署最安全，但成本和技术门槛高。

根据自己的场景，选合适的组合，别把鸡蛋放一个篮子里。

大抵如此罢。
