# 博客主页完整升级设计规格

**日期：** 2026-04-15
**状态：** 已批准，待实现
**目标：** 增强个人品牌感、视觉层次清晰、有温度的用户体验

---

## 1. 设计目标

### 1.1 核心问题
当前主页存在以下问题需要解决：
- **视觉层次不清晰**：信息密度高但缺乏引导，用户不知道先看哪里
- **缺少视觉元素和微交互**：页面感觉"冷"，不够生动

### 1.2 设计原则
- **文章优先**：读者优先级为 文章 > 作者 > 项目案例
- **个人品牌感**：体现"有温度"，像在和真人对话
- **视觉丰富但不过度**：彩色条、图标、动画适度使用

## 4. 微交互设计

### 4.1 卡片悬停

```css
.post-card:hover {
  transform: translateY(-4px) scale(1.02);
  box-shadow: 0 12px 24px rgba(0,0,0,0.1);
  transition: all 0.3s ease;
}
```

### 4.2 标签悬停

```css
.tag:hover {
  filter: brightness(1.1);
  cursor: pointer;
}
```

### 4.3 时间线动画

```svg
<line stroke-dasharray="8 4">
  <animate attributeName="stroke-dashoffset"
           from="500" to="0"
           dur="3s"
           repeatCount="indefinite"/>
</line>
```

### 4.4 进入动画

```css
/* 文章卡片依次淡入 */
.post-card {
  animation: fadeInUp 0.5s ease forwards;
  animation-delay: calc(var(--index) * 100ms);
}

@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

## 5. 响应式设计

### 5.1 断点

- Mobile: < 768px
- Tablet: 768px - 1024px
- Desktop: > 1024px

### 5.2 关键调整

**Mobile (< 768px):**
- Hero：单列布局，动效图移至文字下方或隐藏
- 文章卡片：单列堆叠
- 技术栈矩阵：2x8 网格
- 时间线：简化节点，缩小尺寸

**Tablet (768px - 1024px):**
- Hero：左右布局保持，动效图缩小
- 文章卡片：2列网格
- 技术栈矩阵：4x4 保持
